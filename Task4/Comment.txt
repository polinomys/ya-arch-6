1. Исходя из описания, назначение сервиса osago-aggregator:
- отправка заявок в страховые компании на основании запроса от core-app;
- опрос страховых компаний на предмет наличия информации по заявке;
- передача полученных результатов обратно в core-app;

При этом важно учесть ряд существенных условий:
- максимальное время ожидания решения от страховой компании — 60 секунд;
- важно, чтобы на экране пользователя предложения от каждой страховой компании отображались сразу, как только от нее пришёл ответ;
- взаимодействие со страховыми компаниями только однотипные REST API;

2. Так как важно, чтобы информация поступала пользователю незамедлительно по мере ее получения, то взаимодействие веб-приложения с core-app предлагается организовать на основе WebSocket;

3. Взаимодействие core-app с сервисом osago-aggregator предлагается организовать асинхронное с использованием брокера сообщений.
Таким образом не увеличивать сильно связность.

4. Взаимодействие сервиса osago-aggregator со страховыми компаниями на основе REST API.

5. Описание механизма взаимодействия.
- core-app отправляет в топик заявку на основе информации пользователя с уникальным идентификатором;
- osago-aggregator вычитывает заявку и формирует запросы к страховым компаниями по REST API на основе заявки;
- osago-aggregator периодически опрашивает страховые компании на предмет наличия ответа по заявке;
- osago-aggregator полученные ответы публикует также в брокер сообщений;
- core-app, получив информацию из брокера сообщений, незамедлительно отдает ее пользователю по WebSocket;

6. Наличие собственной БД для сервиса osago-aggregator: Да (osago-aggregator-db).
Как минимум для контроля заявок, по которым необходимо формировать запросы на получение статуса заявки.

7. Применение паттернов отказоустойчивости.
Circuit Breaker - нет;
Timeout - да;
Rate Limiting - да;
Retry - да;
